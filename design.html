Project description

This site is a personal homepage built with HTML5, CSS3, and ES6 modules (no frameworks, no jQuery). It showcases who I am, highlights projects, and provides quick access to my resume. The design aims to be fast, responsive, accessible, and easy to extend.

User personas

Maya — Tech Recruiter

Skims portfolios to fill SWE/Data roles

Goal-oriented
  • Goals: Judge fit fast, grab resume, skim 2–3 projects.
  • Behaviors: 30–60s per site; mobile or laptop; hates slow pages.
  • Pain points: Messy navigation, buried resume, vague project impact.
  • Success: Finds resume in 1 click; sees measurable outcomes on projects.

Dr. Lee — Course Instructor

Evaluates assignment criteria & craft quality

Evaluator
  • Goals: Confirm rubric: ES modules, accessibility, original JS.
  • Behaviors: Tests on desktop; inspects code structure & linting.
  • Pain points: Hidden code, unclear design rationale.
  • Success: Clear design doc, readable code, working interactions.

Arjun — Grad Peer/Collaborator

Looking for teammates & code samples

Peer
  • Goals: Skim tech stack, demos, and contact links.
  • Behaviors: Desktop first; reads project summaries thoroughly.
  • Pain points: No live links, no stack mentions, poor structure.
  • Success: Understands skills + finds a project worth exploring.

User stories

Recruiter — Resume access

As a recruiter, I want to download a PDF resume in one click so that I can attach it to the ATS quickly.

Acceptance criteria
  • Given I’m on any page, when I open the navigation, then I see a link to “Resume”.
  • When I click it, then a PDF is visible with a visible Download link.

Recruiter — Project impact

As a recruiter, I want 2–3 bullets per project with outcomes so that I can assess impact quickly.

Acceptance criteria
  • Each card shows a short description with measurable results or tech used.
  • Cards are readable on mobile and desktop.

Instructor — Evaluate rubric items

As an instructor, I want a dedicated Design Document so that I can verify personas, stories, and mockups quickly.

Acceptance criteria
  • Design page lists project description, personas, stories, and mockups.
  • Site uses ES6 modules; original JS interaction is present (canvas particles).
  • Lint passes with provided config.

Peer — Find contact

As a peer, I want obvious links to GitHub/LinkedIn so that I can reach out for collaboration.

Acceptance criteria
  • Footer contains GitHub/LinkedIn/Email links on every page.

All — Performance & access

As a visitor, I want pages to load fast and be readable in dark or light themes so that the experience is comfortable.

Acceptance criteria
  • Core content loads without blocking JS; theme toggle remembers preference.
  • Text has sufficient contrast; images have alt text.
