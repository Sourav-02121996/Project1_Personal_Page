Team website overview

This site is a lightweight, accessible showcase for our team, built with HTML5, CSS3, and ES6 modules. Each member highlights their strengths with concise personas and user stories that directly inform the content and interactions across pages.

User stories

Aditya — Surface DE stack and outcomes

As a data-engineering recruiter viewing Aditya’s work, I want to see pipeline tags (e.g., Airflow, Spark, Delta Lake) and 1–2 outcome metrics so that I can judge relevance in under 30 seconds.

Aditya — Quick résumé + case study

It will be helpful If recruiters get a persistent résumé link and a short case-study page so that recruiters can download my résumé and skim context without hunting.

Akash — Clear demo & repo paths

As a peer or recruiter, I want obvious “Demo” and “Repo” buttons for Akash’s projects so that I can try or review the code with minimal friction.

Akash — Architecture clarity

Recruiters want each project to include a short architecture note (modules/responsibilities) so that reviewers understand the design quickly.

Skandhan — API discoverability

As a backend-focused reviewer, I want example requests for Skandhan’s APIs so that I can test endpoints immediately.

Skandhan — Performance snapshot

As a hiring manager, I want latency/throughput or model metrics visible so that I can assess performance at a glance.

Siddharth — Insight-first dashboards

As a stakeholder, I want 2–3 insights under each dashboard thumbnail so that I immediately grasp the “so what”.

Siddharth — Sharable artifacts

As a teacher, I want downloadable slide/PDF exports for dashboards so that I can share them quickly with non-technical reviewers.
